### ✅ Task 2：LIFF 非阻塞初始化
**狀態**：✓ 完成

**改動**：
- `src/contexts/LiffContext.tsx` - 改造為非阻塞模式
- 加入 5 秒 timeout 機制（防止無限等待）
- 加入 localStorage 快取（加快恢復速度）
- 後台初始化（不阻塞 UI 渲染）

**檔案**：
- [src/contexts/LiffContext.tsx](src/contexts/LiffContext.tsx) - 完整改造

**預期效果**：-1000ms 初始化延遲 + 優雅 Fallback

**核心改進**：
1. **立即返回 `liffReady = true`** - UI 可以繼續渲染
2. **5 秒 Timeout** - 超時自動 fallback，不卡住
3. **localStorage 快取** - 登入過的用戶 0ms 恢復
4. **並行請求** - getProfile 同時獲取，無額外等待

```tsx
// ❌ 舊版：同步阻塞
liff.init({ liffId })  // 卡在這裡...
  .then(() => {...})

// ✅ 新版：非阻塞
setLiffReady(true)  // 立即返回
initLiffInBackground(liffId)  // 後台進行
```

### ✅ Task 3：stampCount 停止輪詢
**狀態**：✓ 完成

**改動**：
1. `passportUtils.ts` 新增 `emitStampUnlockedEvent()` 函數
2. `passportUtils.ts` - `unlockStamp()` 中調用事件發射
3. `App.tsx` Header - 移除 `setInterval()`，改用事件監聽

**檔案**：
- [passportUtils.ts](passportUtils.ts) - 新增事件機制
- [App.tsx](App.tsx#L56-88) - 改用事件驅動

**預期效果**：移除每秒無謂重渲染

**核心改進**：
1. **事件驅動** - 只在印章解鎖時更新（而不是每秒都檢查）
2. **跨窗口同步** - 也監聽 localStorage 變化（標籤頁同步）
3. **零輪詢開銷** - 完全消除 setInterval

```tsx
// ❌ 舊版：每秒檢查（1000ms 間隔）
setInterval(() => {
  setStampCount(getUnlockedStampCount())  // 每秒執行！
}, 1000)

// ✅ 新版：事件驅動
document.addEventListener('stamp-unlocked', () => {
  setStampCount(getUnlockedStampCount())  // 只在改變時
})
```

## 💡 Q&A

**Q：為什麼改 SVG 後沒看到 logo？**  
A：需要確認 `public/logo.svg` 檔案存在且 Vite 正確配置。如果還是看不到，可以改用 data URL 內嵌。

**Q：LIFF 登入功能還能用嗎？**  
A：能。只是改成後台初始化，UI 先顯示，登入狀態後補。護照功能依然完整。

**Q：stampCount 改事件驅動後，跨標籤頁同步嗎？**  
A：會。同時監聽 `stamp-unlocked` 自訂事件 和 `storage` 事件，確保跨標籤同步。
